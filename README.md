# Study 'N Stack - Flashcard Digital Learning App

[cite_start]Study 'N Stack adalah aplikasi media belajar flashcard digital berbasis Android yang dirancang untuk membantu mahasiswa, khususnya di bidang Teknik Komputer, dalam memahami dan mengingat konsep mata kuliah melalui metode *active recall*[cite: 26, 31].

## 🎯 Masalah & Solusi
* [cite_start]**Masalah:** Mahasiswa sering kesulitan mengingat konsep mata kuliah yang kompleks dan merasa metode belajar tradisional membosankan[cite: 24, 25].
* [cite_start]**Solusi:** Study 'N Stack menyediakan fitur chat interaktif yang meniru asisten belajar untuk membuat proses menghafal lebih aktif, menarik, dan terarah[cite: 26, 27, 29].

---

## ✨ Fitur Utama
* [cite_start]**Chat Interaktif:** Antarmuka percakapan sederhana untuk memilih mata kuliah sebelum mulai belajar[cite: 32, 67].
* [cite_start]**Deck & Card:** Pengaturan materi belajar yang terbagi menjadi kartu pertanyaan dan jawaban[cite: 34].
* [cite_start]**Skor & Level:** Memberikan feedback berupa persentase hasil belajar dan tingkat pemahaman pengguna[cite: 35, 67].
* [cite_start]**Recap Belajar:** Menyimpan riwayat dan menampilkan grafik progres nilai harian secara visual[cite: 36, 67, 1374].
* [cite_start]**Database Lokal (Offline):** Menggunakan Isar/Room Database agar aplikasi ringan, cepat, dan dapat diakses tanpa koneksi internet[cite: 58, 59, 1384].

## 🛠️ Spesifikasi Teknologi
* [cite_start]**Framework:** Flutter [cite: 108]
* [cite_start]**Bahasa Pemrograman:** Dart & JSON [cite: 102, 103]

* [cite_start]**IDE:** VS Code [cite: 110]
* [cite_start]**Hardware Minimum:** Smartphone Android 7.0 (Nougat) dengan RAM 2-3 GB[cite: 96, 97, 98].

## 📂 Struktur Data (OOP)
Aplikasi ini mengimplementasikan prinsip Pemrograman Berorientasi Objek (OOP):
* [cite_start]**Encapsulation:** Logika database dipisahkan dalam class `IsarService`[cite: 1378].
* [cite_start]**Abstraction:** Menyederhanakan interaksi UI melalui komponen seperti `ChatBubble`[cite: 1379].
* [cite_start]**Polymorphism:** Menggunakan `MessageType` untuk menangani berbagai tipe pesan (teks, aksi, typing) dalam satu widget[cite: 1381].
